# 完整知识卡规范

## 成品目标

输出可直接发布的 3:4 知识卡，而不是只有隐喻动作的插画底图。默认让 `gpt-image-2` 直接生成“手绘插画 + 中文讲解 + 公式/代码”的单张最终 PNG；信息密度保持中到高，白色留白用于分组和呼吸，不是让画面空掉。

## 固定分区

```text
标题区 10%-14%：页码、标题、最多一条短副标题
核心讲解区 28%-36%：主图解、角色动作、核心定义或规则
过程与辅助教学区 36%-44%：3–5 个过程、例子、公式、代码、对比或易错点模块
结论区 12%-16%：一句结论、复杂度、记忆卡或面试话术
```

## 内容密度

每页只承载一个可复述的结论，并组合至少四类内容：

- 一个主结论或定义。
- 一个可视化过程、例子或真实界面。
- 2–4 个辅助教学模块：对比、条件、注意点、公式、代码片段、清单或结论。

算法页可使用短公式、4–6 行核心代码、状态表或概率链；需要代码的页面可以同时带一条短公式或一个微型状态图，但避免同页同时塞入长代码、大表格和完整推导。

## 信息密度拒绝线

以下任一情况应拆页或重生，不得硬塞：

- 标题、副标题和正文合计超过 160 个汉字，或有超过 6 个独立文字模块。
- 同页同时出现完整概率推导、长代码块、完整状态表和扩展变体，导致阅读主线消失。
- 主视觉小于画面约 20%，导致卡片像纯文字讲义。
- 正文少于 55 个汉字且没有流程、对比、例子或记忆卡，导致卡片像空封面。
- 有效内容区域不足画面约 60%，或手机缩略图不能在 3 秒内看出“标题—主动作—结论”。

## 文字与图层

- 生图模型必须直接绘制手绘角色、动作、物体、箭头、标题、关键术语、公式、命令、代码和结论；每页的精确文字由提示词提供。
- 文字必须落在安全区，并保持手绘知识卡的统一感；若文字、公式或状态有错误，重生成整页，禁止以半透明覆盖层遮住。
- 只有个别错字且用户明确同意时才允许局部文字修正；只提供 overlay 文案建议不算完成。

## 画面语言

- 黑色手绘线稿、白底、红蓝橙少量强调。
- 原创小煤球占画面约 15%–25%，正在讲解、操作或验证，不抢走内容区。
- 用手绘边框、箭头、对话框、表格格线、编号和小图标组织信息。
- 避免纯插画底图、大片无内容空白、只写一个标题的封面式页面。

## 底图提示词

提示词必须明确主动作、语义锁、阅读顺序、安全区和逐字文字内容，例如：

```text
one dominant hand-drawn algorithm action; render the supplied Chinese title, labels,
formula and short code exactly as written; no placeholder cards, no text overlap,
no cropped text, no invented technical details.
```

技术文字交给生图模型直接绘制；提示词中的“精确文字”是唯一内容源，成图后必须逐项核验。
