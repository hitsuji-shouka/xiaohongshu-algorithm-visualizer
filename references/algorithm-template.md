# 算法图解模板

## 分页字段

每页都输出：

```text
页码：
页面目的：
主标题：
一句话结论：
辅助信息：
视觉认知锚点：
角色动作：
建议短标签：
必须准确的技术文字：
生图提示词：
```

## 叙事原则

- 先讲问题，不先讲代码。
- 先用小例子，再给抽象规则。
- 正确性解释尽量用直觉 + 一条关键公式或不变量。
- 在小例子后，默认增加一页真实算法题：题目必须说明输入限制、算法映射和 4–6 行核心代码，不只给抽象伪代码。
- 实战题页只展示核心片段，不放满屏代码；题意、关键条件和代码必须相互对应。
- 优先选择与主题天然匹配的经典题目。例如蓄水池抽样可用“链表随机节点”：链表长度未知，只能单次遍历，`1/i` 替换规则正好成立。
- 总结页要适合收藏和面试前复习。

## 视觉原则

- 用物理动作表现算法状态变化。
- 角色要推动、替换、选择、比较、排队或记录。
- 强调色只标出当前元素、候选答案、关键概率、复杂度结论。

## 标题方向

- 面试高频：`XXX 一次讲懂`
- 反直觉：`为什么后来的数据也能公平选中？`
- 问题型：`不知道总数，怎么随机抽样？`
- 收藏型：`一页记住 XXX 的核心思路`
