# 成图渲染与验收合同

## 页面语义合同

每页在生图前记录：

| 字段 | 要求 |
| --- | --- |
| 核心结论 | 一句可验证的结论 |
| 语义锁 | 样本数、索引体系、变量、允许公式 |
| 主动作 | 角色在做的算法动作 |
| 精确文字 | 标题、正文、公式、代码的唯一来源 |
| 安全区 | 顶部标题区、底部教学区、不可覆盖的主视觉区 |

不要从不同页复用含变量、座位、公式或状态箭头的底图。

## 底图检查

用 `view_image` 在合成前检查每张底图：

- 白底、手绘线条、角色动作和颜色是否符合主题。
- 是否完全无技术文字、无数字、无代码、无公式、无伪表格。
- 主视觉是否与该页语义锁一致，例如单样本只有一个候选位。
- 是否是黑色实心小黑、白点眼睛和细短腿；出现火柴人、白脸线稿人或普通人类角色时重生成。
- 标题和教学安全区是否干净；若不干净，重生成，不要遮盖。

## 排版检查

- 逐字从“精确文字”字段渲染；中文使用 CJK 字体，代码使用等宽字体。
- 公式与代码使用独立框，不自动拆词；代码保留 4–8 行核心实现。
- 以边界框验证：任何两块文字、文字与箭头、文字与关键状态的间距至少 16 px。
- 导出前确认每个页面有：主结论、主图解、至少一个辅助教学模块和一句结论/易错点。

## 最终验收

1. 用源文本逐条核对变量、索引、概率、公式、复杂度和样本数。
2. 在 100% 尺寸检查中文、公式、代码、边距和覆盖关系。
3. 在手机缩略图检查标题—主动作—结论的阅读顺序。
4. 用像素工具检查每张 PNG 精确为 1080 × 1440。
5. 任意不通过：先修底图或排版，再重做本页验收；不得仅修改发布文案后放行。
