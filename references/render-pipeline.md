# 成图渲染与验收合同

## 页面语义合同

每页在生图前记录：

| 字段 | 要求 |
| --- | --- |
| 核心结论 | 一句可验证的结论 |
| 语义锁 | 样本数、索引体系、变量、允许公式 |
| 主动作 | 角色在做的算法动作 |
| 精确文字 | 标题、正文、公式、代码的唯一来源 |
| 安全区 | 顶部标题区、底部教学区、不可覆盖的主视觉区 |

不要从不同页复用含变量、座位、公式或状态箭头的底图。

## 完整卡检查

用 `view_image` 检查每张 `image_gen` 直接生成的完整卡：

- 白底、手绘线条、角色动作和颜色是否符合主题。
- 是否完整呈现“精确文字”字段中的标题、短解说、公式、代码、页码和结论。
- 主视觉是否与该页语义锁一致，例如单样本只有一个候选位。
- 是否是黑色蓬松小煤球、偏白大圆眼睛和细短腿；出现火柴人、白脸线稿人或普通人类角色时重生成。
- 是否有文字重叠、裁切、乱码、错字、空框或占位横线；有任一问题就重生成，不要遮盖。

## 可选局部修正检查

- 只用于用户明确要求修一个错字，或连续重生后仍无法获得正确文本的情形。
- 保留模型生成的原始构图和手绘感，不得把整张卡变成后期排版海报。
- 导出前确认每个页面有：主结论、主图解、至少一个辅助教学模块和一句结论/易错点。

## 最终验收

1. 用“精确文字”逐条核对模型生成的变量、索引、概率、公式、复杂度和样本数。
2. 在 100% 尺寸检查中文、公式、代码、边距和覆盖关系；出现错字或乱码则重生。
3. 在手机缩略图检查标题—主动作—结论的阅读顺序。
4. 用像素工具检查每张 PNG 精确为 1080 × 1440。
5. 任意不通过：优先重生整页；只有个别文字问题且用户同意时才局部修正，不得仅修改发布文案后放行。
